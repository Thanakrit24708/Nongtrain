# รายงานโครงการพัฒนาเว็บแอปพลิเคชันเพื่อสุขภาพ "NongTrain"

---

## 1. ที่มาและความสำคัญ

ในยุคปัจจุบัน ผู้คนหันมาใส่ใจสุขภาพมากขึ้น การควบคุมอาหารและการออกกำลังกายเป็นหัวใจสำคัญของการมีสุขภาพที่ดี อย่างไรก็ตาม การคำนวณพลังงาน (Calorie) และจดบันทึกโภชนาการในแต่ละวันทำได้ยากหากไม่มีเครื่องมือช่วย โปรเจกต์ "NongTrain" จึงถูกพัฒนาขึ้นเพื่อเป็นเครื่องมือช่วยในการติดตามสุขภาพ คำนวณพลังงาน และวางแผนการออกกำลังกายที่ง่ายต่อการใช้งานและเข้าถึงได้ทุกที่ผ่านหน้าเว็บ

## 2. วัตถุประสงค์ของโครงการ

1. เพื่อพัฒนา Web Application สำหรับติดตามโภชนาการและบันทึกการกิน (Food Tracking)
2. เพื่อให้ผู้ใช้สามารถคำนวณแคลอรี่ที่เหมาะสมต่อวัน (TDEE/BMR) โดยอัตโนมัติ
3. เพื่อสร้างแรงจูงใจในการดูแลสุขภาพผ่านการแสดงผลกราฟและสถิติที่เข้าใจง่าย
4. เพื่อศึกษาการใช้งาน HTML, CSS, JavaScript ในการสร้าง Interactive Dashboard

## 3. กลุ่มเป้าหมาย

- บุคคลทั่วไปที่ต้องการลดน้ำหนัก, รักษารูปร่าง หรือเพิ่มกล้ามเนื้อ
- ผู้ที่ออกกำลังกายและต้องการบันทึกแคลอรี่
- ผู้เริ่มต้นที่ต้องการตัวช่วยคำนวณโภชนาการ

## 4. ขอบเขตและฟีเจอร์การทำงาน (Features Scope)

ระบบ NongTrain ประกอบด้วยฟังก์ชันหลักดังนี้:

### 4.1 ระบบ Onboarding และ Profile

- ลงทะเบียนข้อมูลส่วนตัว (ชื่อ, อายุ, เพศ, ส่วนสูง, น้ำหนัก)
- เลือกเป้าหมายสุขภาพ (ลดน้ำหนัก / Maintain / เพิ่มกล้ามเนื้อ)
- คำนวณค่า TDEE (Total Daily Energy Expenditure) และตั้งเป้าหมายแคลอรี่ต่อวันอัตโนมัติ

### 4.2 ระบบบันทึกอาหาร (Smart Food Tracking)

- บันทึกรายการอาหารพร้อมปริมาณแคลอรี่ โปรตีน คาร์โบไฮเดรต และไขมัน
- **Thai Food Database**: มีฐานข้อมูลอาหารไทยยอดนิยมให้ค้นหาและเติมข้อมูลอัตโนมัติ
- คำนวณยอดแคลอรี่คงเหลือแบบ Real-time (Target - Eating)

### 4.3 ระบบบันทึกการออกกำลังกาย (Activity Logging)

- บันทึกชื่อกิจกรรมและระยะเวลา
- คำนวณแคลอรี่ที่เผาผลาญ (Burned Calories)
- แสดงแยกเป็น "พลังงานขาเข้า" (In) และ "พลังงานขาออก" (Out) อย่างชัดเจน
- **Exercise Recommendation**: ระบบแนะนำสัดส่วนการออกกำลังกาย (Cardio vs Weight) ให้เหมาะสมกับเป้าหมาย

### 4.4 ระบบแสดงผลและสถิติ (Visual Analytics)

- **Dashboard**: แสดงวงกลมความคืบหน้า (Circular Progress) ของแคลอรี่ที่กินไป
- **Macro Bars**: กราฟแท่งแสดงสัดส่วนสารอาหาร (Protein, Carb, Fat)
- **Weight Chart**: กราฟเส้นแสดงพัฒนาการน้ำหนักตัวตามช่วงเวลา
- **BMI Status**: คำนวณดัชนีมวลกายพร้อมคำแนะนำ (ผอม, ปกติ, อ้วน)

## 5. โครงสร้างระบบ (System Architecture)

ระบบพัฒนาด้วยสถาปัตยกรรมแบบ Single Page Application (SPA) อย่างง่าย โดยใช้ Client-Side Rendering ทั้งหมด

- **Front-end**: HTML5, CSS3, JavaScript (ES6)
- **Data Persistence**: ใช้ `localStorage` ของ Browser ในการเก็บข้อมูลผู้ใช้ (User Profile, Food Logs, Exercise Logs) ทำให้ข้อมูลยังอยู่แม้ปิดหน้าเว็บไปแล้ว โดยไม่ต้องใช้ Backend Server
- **External Libraries**: `Chart.js` สำหรับการสร้างกราฟ

### ไฟล์สำคัญในโปรเจกต์

- `index.html`: โครงสร้างหน้าเว็บหลัก แบ่ง Section ต่างๆ ด้วย `<section id="...">`
- `style.css`: จัดการดีไซน์ใช้ Theme สีสะอาดตา (Glassmorphism), Responsive Design รองรับมือถือ
- `script.js`: จัดการลอจิกทั้งหมด (การคำนวณ, การบันทึก, การแสดงผล DOM Manipulation)

## 6. เครื่องมือที่ใช้พัฒนา (Tools)

- Code Editor: VS Code
- Browser: Google Chrome / Microsoft Edge (สำหรับทดสอบ)
- Image Generation AI: Pollinations.ai (สำหรับสร้างรูปประกอบเมนูอาหาร - Experimental)

## 7. สรุปผลการดำเนินงาน

โปรเจกต์ NongTrain สามารถทำงานได้ตามวัตถุประสงค์ที่ตั้งไว้ ผู้ใช้สามารถบันทึกข้อมูลโภชนาการและดูสรุปผลผ่าน Dashboard ได้อย่างรวดเร็ว ระบบมีความเสถียรในการใช้งานบน Browser ทั่วไป และมีการจัดการ Error Handling ที่ดี (เช่น การแจ้งเตือนเมื่อกรอกข้อมูลผิดพลาด)

## 8. ข้อเสนอแนะและแผนพัฒนาต่อ (Future Work)

- พัฒนาระบบ Login เชื่อมต่อ Database จริง (Firebase/MySQL) เพื่อให้ใช้งานได้หลายอุปกรณ์
- เพิ่มฟีเจอร์ Social Sharing แชร์ผลลัพธ์ลง Social Media
- เชื่อมต่อ API ฐานข้อมูลโภชนาการภายนอกเพื่อให้มีรายการอาหารหลากหลายขึ้น
